# Security Policy

## Scope

This policy applies to the LLM Consensus System (LCS) codebase and
infrastructure managed in this repository.

## Reporting a Vulnerability

Do not report security vulnerabilities through public GitHub issues.

Send reports to the security contact listed below.

Security contact: remiboivin021@gmail.com

### What to Include

- Type of vulnerability
- Affected components and file paths
- Location of affected code (tag/branch/commit)
- Steps to reproduce
- Proof-of-concept, if available
- Potential impact
- Possible mitigations

### Response Targets (Phase A)

- Acknowledgment within 72 hours
- Initial assessment within 7 days
- Fix timeline is best-effort, based on severity

## Disclosure Policy

- Vulnerabilities must be reported privately.
- Public disclosure happens after a fix is available.
- Coordinated disclosure timelines will be agreed per case.

## Migration to Phase B

Once the product is declared ready, the project will adopt the standard
security process (Phase B) with:

- GitHub Security Advisories as the primary channel
- Public advisories for confirmed vulnerabilities
- A 48-hour acknowledgment target
- A 30-day target for fixes on supported versions

## Supported Versions

Security fixes are provided for the latest released version.

## Safe Harbor

We will not take legal action against researchers who:

- Act in good faith
- Avoid accessing, modifying, or deleting user data
- Report issues responsibly
- Do not disclose the vulnerability before a fix is available
